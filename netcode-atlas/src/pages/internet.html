Internet Protocols

The rules and standards that govern communication across the internet

What are Internet Protocols?

Internet protocols are sets of rules that define how data is transmitted and received over networks. They ensure that devices can communicate with each other regardless of their underlying hardware or software.

The internet protocol suite, commonly known as TCP/IP, provides end-to-end data communication specifying how data should be packetized, addressed, transmitted, routed, and received.

Protocol Adoption Over Time

The internet has evolved significantly, with new protocols being developed and adopted to improve security, performance, and capabilities:

Key Internet Protocols

HTTP/HTTPS

Hypertext Transfer Protocol (Secure) is the foundation of data communication for the World Wide Web.

GET /index.html HTTP/1.1 Host: www.example.com

TCP

Transmission Control Protocol provides reliable, ordered, and error-checked delivery of data.

Source Port: 54321 Destination Port: 80 Sequence Number: 1234567890

UDP

User Datagram Protocol provides connectionless communication with minimal protocol mechanism.

Source Port: 12345 Destination Port: 53 Length: 512 bytes

IP

Internet Protocol delivers packets from the source host to the destination host based on IP addresses.

Source: 192.168.1.100 Destination: 93.184.216.34 TTL: 64

HTTP Evolution

HTTP/1.0 (1996)

The first protocol version to specify version numbers. Each request requires a separate TCP connection.

HTTP/1.1 (1997)

Introduced persistent connections, chunked transfer encoding, and additional cache mechanisms.

HTTP/2 (2015)

Major revision that introduced multiplexing, header compression, and server push capabilities.

HTTP/3 (2022)

Uses QUIC transport protocol over UDP instead of TCP, reducing connection establishment latency.

Security Protocols

TLS/SSL

Transport Layer Security and its predecessor Secure Sockets Layer provide communications security over a computer network.

HTTPS

HTTP Secure is the combination of HTTP with TLS to provide encrypted communication and secure identification.

DNSSEC

DNS Security Extensions add security to the Domain Name System by enabling cryptographic authentication.

SSH

Secure Shell provides a secure channel over an unsecured network for remote login and other secure network services.
