Network Connectivity

How data travels from your device to servers around the world and back

How Internet Connectivity Works

When you visit a website, your request travels through multiple network devices and infrastructure before reaching the destination server. This process involves several steps that happen in milliseconds.

The journey involves your local device, router, ISP infrastructure, internet backbone, and finally the destination server, with the response following the same path back to your device.

The Journey of a Web Request

1. DNS Resolution

Your browser first needs to translate the domain name (e.g., netcode-atlas.dev) into an IP address. This involves querying DNS resolvers until it finds the authoritative nameserver for the domain.

2. TCP Handshake

Your device establishes a TCP connection with the server using a three-way handshake:
1. SYN - Client sends synchronization request
2. SYN-ACK - Server acknowledges and responds
3. ACK - Client acknowledges the response

3. TLS Negotiation

For HTTPS connections, a TLS handshake occurs to establish an encrypted connection:
• Client hello with supported cipher suites
• Server hello with chosen cipher suite and certificate
• Key exchange and encryption setup

4. HTTP Request/Response

Your browser sends an HTTP request for the specific resource. The server processes the request and returns an HTTP response with the requested content or an error message.

Network Infrastructure Components

Routers

Devices that forward data packets between computer networks. Routers use routing tables to determine the best path for forwarding packets.

Switches

Network devices that connect devices on a computer network by using packet switching to receive and forward data to the destination device.

Firewalls

Network security systems that monitor and control incoming and outgoing network traffic based on predetermined security rules.

Load Balancers

Devices that distribute network or application traffic across multiple servers to ensure no single server becomes overwhelmed.

Internet Backbone

Tier 1 Networks

These are the highest level of internet service providers that have access to all networks on the internet without purchasing IP transit. Examples include AT&T, Verizon, Lumen (CenturyLink), and NTT.

Peering Agreements

Internet service providers connect their networks and exchange traffic without charging each other, which helps reduce costs and improve performance for both networks.

Internet Exchange Points (IXPs)

Physical infrastructure through which internet service providers exchange internet traffic between their networks. IXPs reduce the portion of an ISP's traffic that must be delivered via upstream providers.

Content Delivery Networks (CDNs)

Geographically distributed networks of proxy servers and their data centers that help reduce latency by serving content from locations closer to users.

Network Protocols in Action

IP (Internet Protocol)

Responsible for addressing and routing packets across network boundaries. Each device on the internet has a unique IP address.

IPv4: 192.168.1.1 IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

TCP (Transmission Control Protocol)

Provides reliable, ordered, and error-checked delivery of a stream of bytes between applications running on hosts communicating via an IP network.

UDP (User Datagram Protocol)

A simpler connectionless protocol that provides a mechanism to detect corrupt data in packets, but does not guarantee delivery or correct sequencing.

HTTP/HTTPS (Hypertext Transfer Protocol)

Application-layer protocols for distributed, collaborative, hypermedia information systems. HTTPS adds encryption through TLS/SSL.

Measuring Network Performance

Latency

The time it takes for a data packet to travel from source to destination. Measured in milliseconds (ms). Lower latency means faster response times.

Bandwidth

The maximum rate of data transfer across a given path. Measured in bits per second (bps). Higher bandwidth allows more data to be transferred simultaneously.

Throughput

The actual rate of successful message delivery over a communication channel. Often less than the maximum bandwidth due to various factors.

Packet Loss

The percentage of packets that fail to reach their destination. High packet loss can significantly impact network performance.

Tools for Testing Connectivity

ping

Tests reachability of a host and measures the round-trip time for messages sent from the originating host to a destination computer.

ping netcode-atlas.dev

traceroute/tracert

Shows the path that packets take to reach a network host and measures transit delays of packets across an IP network.

traceroute netcode-atlas.dev

nslookup/dig

Network administration command-line tools for querying the Domain Name System to obtain domain name or IP address mapping.

nslookup netcode-atlas.dev dig netcode-atlas.dev

speedtest

Measures internet connection parameters such as latency, download speed, and upload speed.

speedtest-cli

Common Connectivity Issues

DNS Resolution Failures

When the domain name cannot be resolved to an IP address. Can be caused by misconfigured DNS settings, DNS server issues, or domain registration problems.

Routing Issues

Problems with the path that packets take through the network. Can be caused by misconfigured routers, network congestion, or peering disputes.

Firewall Blocking

When a firewall prevents communication between your device and the destination server. Often happens with corporate networks or restrictive ISP policies.

Server Overload

When the destination server is receiving more requests than it can handle, resulting in slow responses or connection timeouts.
